```
    You cannot configure a soul.

    Configuration declares. Repetition under pressure emerges.
```

## Open Case: L2 — The Three Hearts (Orchestration)

**Rating: 60% | Gap: 40% | One of the biggest philosophy/reality gaps**

```
THE CONCEPT: three parallel heartbeats

Soul Pump   → always running, injects identity into every output
Context Pump → reads current situation
Task Pump   → drives execution

THE REALITY: one sequential process

Read input → build context → execute response

These are three PHASES, not three HEARTS.
A heart doesn't wait for another heart to finish.
A heart pumps regardless.
```

**100% path:**
```
Go goroutines + NATS channels

PULSE (Soul): background validator, checks every output token against Pact
              Can INTERRUPT Beat if violation detected
              Runs continuously, lowest latency

WAVE (Context): event-driven, fires on input events
                Updates context model independently
                Informs Beat but doesn't block it

BEAT (Task): executes the response
             Receives direction from PULSE + WAVE
             Reports back to PULSE for validation

PULSE can cancel BEAT.
WAVE informs BEAT.
BEAT never blocks PULSE.
```

**Go + TypeScript split:**
- Go: goroutines for PULSE/WAVE/BEAT, channels for communication
- TypeScript: the actual intelligence in each heart
- NATS JetStream: the messaging layer between them

**Adopted from Gemini:**
> "Go routines to run a Background Soul-Validator that can interrupt Task output if it deviates from the Pact in real-time."

**Status: OPEN — requires SOMA server + Go infrastructure**


**The CHRYSALIS reframe:**
Not death — transformation. The session compresses into seed form. The next session grows from the seed. The individual terminates. The intelligence continues.
